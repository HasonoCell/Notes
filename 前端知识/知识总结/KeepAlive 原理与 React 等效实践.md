## 一、 Vue `<KeepAlive>` 简介与设计哲学

`<KeepAlive>` 是 Vue 内置的一个**抽象组件**（自身不渲染为真实 DOM）。它的核心作用是包裹动态组件时，缓存不活动的组件实例，而不是直接销毁它们。

**设计哲学：**

- **务实主义**：Vue 关注解决前端工程的实际痛点（如 Tab 切换、路由跳转时的状态丢失和卡顿）。
    
- **空间换时间**：牺牲部分内存（RAM）来缓存组件实例和 DOM 树，换取 CPU 零负荷的极致切换速度。
    
- **基于对象的组件模型**：Vue 组件底层是响应式对象实例，这使得“脱离 DOM 树但保留内存状态”成为可能，天然契合缓存机制。
    

## 二、 底层原理：状态保留与 DOM 级性能优化

在处理包含海量 DOM（如复杂图表、大型数据表格）的组件切换时，`<KeepAlive>` 提供的是**物理级的性能优化**。

### 1. 为什么能优化 DOM 增删？($O(N)$ -> $O(1)$)

如果没有 `<KeepAlive>`，组件切换的代价是极其昂贵的 `$O(N)$` 操作：

> 解析 VNode -> 创建真实 DOM 树 -> 插入页面 -> 卸载 -> 销毁真实 DOM 树 -> 垃圾回收 (GC)

**包裹 `<KeepAlive>` 后的底层核心动作：**

- **离开时（缓存）**：Vue 不会销毁组件的 VNode 和真实 DOM。它会在底层调用类似于原生 `parentNode.removeChild(el)` 的 API，将整棵庞大的真实 DOM 树从页面的可视区域中“拔”出来，并将其引用存储在 `<KeepAlive>` 内部的 JavaScript `Map` 对象中。此时，DOM 节点脱离了文档流，但依然驻留在内存里。
    
- **进入时（复用）**：Vue 会跳过组件初始化、VNode Diff 和 DOM 创建的全过程。它直接从内存中取出之前缓存的真实 DOM 树，调用类似于 `parentNode.appendChild(el)` 或 `insertBefore` 的原生 API，将其原封不动地“塞”回页面中。
    

通过简单的 `removeChild` 和 `appendChild` 级别的节点搬运，DOM 切换的计算开销被直接降维到了 `$O(1)$`。

### 2. 状态是如何保留的？

因为组件的 Vue 实例（包含 data、computed 等响应式状态）被一同缓存在了内存中，实例没有被销毁，所以之前填写的表单数据、滚动条位置等状态自然得以完美保留。

### 3. 防范内存泄漏：LRU 算法

为了防止无限缓存导致浏览器 OOM（内存溢出），`<KeepAlive>` 内部实现了 **LRU（最近最少使用）缓存淘汰算法**。

- 通过配置 `max` 属性（如 `<KeepAlive max="10">`）。
    
- 当缓存数量超过设定的阈值时，Vue 会自动找到**最久没有被访问过**的那个组件实例，真正执行它的销毁逻辑并释放 DOM 内存。

## 三、 React 的哲学限制与目前最佳实践

React 的核心哲学是 **$UI = f(state)$（纯函数与不可变性）**。组件的挂载/卸载与状态的存亡是强绑定的。因此，React 官方长久以来没有提供全局的 KeepAlive 组件。

但在实际业务中，为了实现类似的 DOM 缓存与状态保留，目前 React 社区有以下几种主流方案：

### 1. 社区最佳实践：`react-activation` (利用 Portals 离屏渲染)

这是目前还原 Vue `<KeepAlive>` 行为最完美、工程化最成熟的第三方库。

- **底层原理**：
    
    1. 在应用的顶层创建一个隐藏的“离屏容器”（Offscreen Container）。
        
    2. 需要缓存的组件始终在这个离屏容器中挂载和渲染（保证其 React 实例和内部状态永远不被销毁）。
        
    3. 利用 React 的 `ReactDOM.createPortal` API，将离屏容器中渲染好的真实 DOM，“传送”到页面上实际需要展示的占位节点中。
        
- **优势**：完美模拟了 DOM 的搬运逻辑，保留了完整的局部状态，并且提供了 `useActivate` 和 `useUnactivate` 生命周期钩子（等效于 Vue 的 `activated` / `deactivated`）。
    
### 2. 官方未来方案：`<Offscreen>` / Activity API

- **现状**：React 核心团队已在 Concurrent 模式下开发了底层的 `<Offscreen>` API（未来稳定版可能命名为 Activity）。
    
- **原理**：允许 React 在后台渲染组件，并在隐藏时将其“挂起”。它保留了状态和 DOM 树，但将该组件的更新优先级降至最低，使其不再参与主线程的计算，从而在遵守 React 哲学的前提下实现了原生级别的性能优化。
